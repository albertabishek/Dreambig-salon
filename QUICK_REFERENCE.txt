================================================================================
                          COLOR FORMAT (OKLch)
================================================================================

Format: oklch(L C h)
  L = Lightness    (0-1)      0=black, 0.5=medium, 1=white
  C = Chroma       (0-0.4)    0=gray, 0.18=vivid
  h = Hue          (0-360)    0=red, 50=gold, 130=green, 250=blue, 280=purple

Examples:
  oklch(0.08 0.04 280) = Dark navy (L=0.08, h=280)
  oklch(0.65 0.18 50)  = Bright gold (L=0.65, C=0.18, h=50)
  oklch(0.98 0 0)      = Off-white (L=0.98, no color)

QUICK PALETTE SUGGESTION:
  Light color:     Keep L above 0.50
  Dark color:      Keep L below 0.25
  Vibrant:         Use C around 0.15-0.20
  Muted:           Use C around 0.05-0.10

================================================================================
                      TROUBLESHOOTING
================================================================================

COLORS DON'T UPDATE:
  1. Make sure you edited app/globals.css
  2. Save the file
  3. Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)
  4. Check if both :root and .dark sections are updated

BUTTON TEXT NOT VISIBLE:
  1. Check the contrast between button color and text color
  2. Verify both :root and .dark sections have proper colors
  3. See COLOR_PALETTE.md for contrast guidelines

DARK MODE LOOKS WRONG:
  1. Check .dark section values differ from :root
  2. Ensure L (lightness) values are appropriate
  3. Test in dark mode browser setting

CAN'T FIND A COLOR:
  1. Search for token name in COLOR_PALETTE.md
  2. Look in app/globals.css :root or .dark section
  3. Check ARCHITECTURE_GUIDE.md for usage

   --button-primary-hover: oklch(0.6 0.16 50);  → oklch(0.6 0.16 250);
